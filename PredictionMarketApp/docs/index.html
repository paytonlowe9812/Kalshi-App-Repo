Install from a public download

These steps work for anyone with a normal browser. No GitHub account is needed as long as the repository is Public.

Maintainers: Replace YOUR-GITHUB-USERNAME with your GitHub user or organization. Replace YOUR-REPO-NAME with the repository name (for github.com/you/Kalshi-App-Repo, use Kalshi-App-Repo). If your default branch is not main, change main in the download URLs.

The source and release ZIPs contain the same tracked files as a git clone (no node_modules, no .venv, no database).

Enable GitHub Pages (maintainers)

  1. Repository Settings → Pages.
  2. Under Build and deployment, set Source to GitHub Actions.
  3. For a monorepo, .github/workflows/pages.yml must sit at the repository root (next to the PredictionMarketApp folder). Push to main or run Deploy Pages. The site appears at https://YOUR-GITHUB-USERNAME.github.io/YOUR-REPO-NAME/.

Make the repository public (maintainers)

  1. On GitHub: Settings → General → Danger Zone → Change repository visibility → Public.
  2. Push a version tag once so the latest release link works, for example: 
    git tag v1.0.0
git push origin v1.0.0
    The release workflow attaches PredictionMarketApp.zip. After that, releases/latest/download/PredictionMarketApp.zip always points at the newest release.

After downloading

  1. Unzip the file.
  2. Open the folder that contains backend, frontend, and requirements.txt. If you unzipped the full repo, open the PredictionMarketApp folder inside it. Release ZIPs usually contain a PredictionMarketApp path at the top — use that folder as the project root.

What you need installed

  • Python 3.10+ — python.org
  • Node.js 18+ LTS and npm — nodejs.org

Install dependencies

Open a terminal in the project root (the folder with requirements.txt).

Windows (PowerShell)

python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
cd frontend
npm install
cd ..

macOS / Linux

python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
cd frontend
npm install
cd ..

The npm install step (above) runs on Windows, macOS, and Linux and automatically creates launch.ps1, launch.bat, and launch.sh in the project root (not in Git). If Python was not on PATH, fix that and run npm install again in frontend/, or run python scripts/generate_launchers.py manually.

Run the app (two terminals)

Keep the virtual environment activated. From the project root:

Terminal 1 — API

python -m uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000

On macOS/Linux use python3 if that is your command for Python 3.

Terminal 2 — web UI

cd frontend
npm run dev

Open http://127.0.0.1:5173 in a browser.

On Windows you can run .\launch.ps1 or launch.bat; on macOS/Linux run ./launch.sh (created during npm install).
