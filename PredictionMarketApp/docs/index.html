Install from a public download

These steps work for anyone with a normal browser. No GitHub account is needed as long as the repository is Public.

Download

On GitHub open your repository, then Releases → Latest. Download PredictionMarketApp-latest-Windows.zip, PredictionMarketApp-latest-macOS.tar.gz, or PredictionMarketApp-latest-Linux.tar.gz for your system. That single release is rebuilt automatically on every push to main; archives match a git clone of the app (no node_modules, no .venv, no database). Release archives intentionally omit launch.ps1, launch.bat, and launch.sh; run the installer or python scripts/generate_launchers.py after unpacking to create them.

Enable GitHub Pages (maintainers)

  1. Repository Settings → Pages.
  2. Under Build and deployment, set Source to GitHub Actions.
  3. For a monorepo, .github/workflows/pages.yml must sit at the repository root (next to the PredictionMarketApp folder). Push to main or run Deploy Pages. GitHub shows the live Pages URL under Settings → Pages after the first successful deploy.

Make the repository public (maintainers)

  1. On GitHub: Settings → General → Danger Zone → Change repository visibility → Public.
  2. Push to main: the workflow .github/workflows/release-zip.yml creates or updates one Latest release (tag latest) with Windows (.zip), macOS (.tar.gz), and Linux (.tar.gz) archives. Each includes a one-line PredictionMarketApp/.release-platform file so GitHub keeps all three downloads. Older semver releases are removed each run so only Latest remains.

After downloading

  1. Unzip the archive.
  2. Open the folder that contains backend, frontend, and requirements.txt (often named PredictionMarketApp).

Prerequisites

  • Python 3.10+ — python.org
  • Node.js 18+ LTS and npm — nodejs.org

Install (one step)

Open a terminal in that project folder, then:

Windows: double-click install.bat, or run .\install.ps1 in PowerShell.

macOS / Linux: chmod +x install.sh once, then ./install.sh.

Start the app

From the same folder:

Windows: launch.bat or .\launch.ps1

macOS / Linux: ./launch.sh

Open http://127.0.0.1:5173.
