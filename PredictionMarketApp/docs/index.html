Install from a public download

These steps work for anyone with a normal browser. No GitHub account is needed as long as the repository is Public.

Maintainers: Replace YOUR-GITHUB-USERNAME everywhere below with your GitHub user or organization (for example octocat). If the repository name is not PredictionMarketApp, replace that too. If your default branch is not main, change main in the download URLs.

The source and release ZIPs contain the same tracked files as a git clone (no node_modules, no .venv, no database).

Enable GitHub Pages (maintainers)

  1. Repository Settings → Pages.
  2. Under Build and deployment, set Source to GitHub Actions.
  3. Push this docs/ site to main (or run the Deploy Pages workflow manually). The site appears at https://YOUR-GITHUB-USERNAME.github.io/PredictionMarketApp/ (use your real username and repo name).

Make the repository public (maintainers)

  1. On GitHub: Settings → General → Danger Zone → Change repository visibility → Public.
  2. Push a version tag once so the latest release link works, for example: 
    git tag v1.0.0
git push origin v1.0.0
    The release workflow attaches PredictionMarketApp.zip. After that, releases/latest/download/PredictionMarketApp.zip always points at the newest release.

After downloading

  1. Unzip the file.
  2. Open the folder that contains backend, frontend, and requirements.txt. GitHub names the folder like PredictionMarketApp-main — that is fine; use it as the project root.

What you need installed

  • Python 3.10+ — python.org
  • Node.js 18+ LTS and npm — nodejs.org

Install dependencies

Open a terminal in the project root (the folder with requirements.txt).

Windows (PowerShell)

python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
cd frontend
npm install
cd ..

macOS / Linux

python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
cd frontend
npm install
cd ..

Run the app (two terminals)

Keep the virtual environment activated. From the project root:

Terminal 1 — API

python -m uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000

On macOS/Linux use python3 if that is your command for Python 3.

Terminal 2 — web UI

cd frontend
npm run dev

Open http://127.0.0.1:5173 in a browser.

On Windows you can instead run .\launch.ps1 from the project root (starts both servers and opens the browser). You may need to allow script execution in PowerShell.
